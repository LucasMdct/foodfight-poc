# FoodFight v0.0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Evolve the `foodfight-poc` into the v0.0.1 game defined by the Guia de Estilo — Skia-rendered food heroes with a walk cycle, the Barão Brigadeiro villain, spinning candy projectiles, a kitchen-table scenario, character selection, a 3-lives model, redesigned HUD and game over, an on-brand loading screen, landscape lock, and centralized (white-label-ready) theming.

**Architecture:** Preserve the POC's performance invariant — one Skia `Canvas`, the gameplay loop on the UI thread via Reanimated shared values, **zero React re-render per frame**. Discrete state (screen, character, lives, score, best) stays in Zustand. Characters and candies are decomposed into Skia primitives; walk-cycle/spin animations are driven by `withRepeat` shared values read through `useDerivedValue` into Skia `Group` transforms. All colors/fonts/radii come from a single `src/theme/` source.

**Tech Stack:** Expo SDK 56, React Native 0.85, TypeScript, `@shopify/react-native-skia` 2.6.2, `react-native-reanimated` 4.3.1, `react-native-gesture-handler`, `zustand`, `expo-font` + `@expo-google-fonts/fredoka`, `expo-screen-orientation`.

## Global Constraints

- **Expo docs:** Before writing code that touches an Expo/RN API, consult `https://docs.expo.dev/versions/v56.0.0/` (per `AGENTS.md`). Expo has changed — do not rely on memory.
- **Performance invariant:** Gameplay must not trigger React re-renders per frame. Per-frame values live in Reanimated shared values read by Skia. Zustand receives only discrete events (hit, game over, one FPS sample/sec).
- **Rendering:** All characters, candies, and scenario render in Skia. No `react-native-svg`.
- **Theming / white-label:** No color, radius, or font literal may appear outside `src/theme/`. Every component reads tokens via `useTheme()` (RN chrome) or receives them as props (Skia draws).
- **Orientation:** The game runs in **landscape**, locked programmatically at boot. There is no orientation-choice screen.
- **Lives model:** Heroes start with **3 lives**; a hit removes 1 with a **1400ms** invulnerability flicker. Game over at 0.
- **Fonts:** Fredoka weights map to families `Fredoka_400Regular`, `Fredoka_500Medium`, `Fredoka_600SemiBold`, `Fredoka_700Bold`.
- **Palette (verbatim from spec):** brand `#FF6B6B` / shadow `#E04F4F`; positive `#7BC950` / shadow `#59A335`; score `#FFB627` / shadow `#E09A10`; villain `#9B5DE5`; candy `#FF7BAC` / `#4ECDC4`; bg `#FFF3E2`, grad `#FFE9C9`→`#FFEFD6`; surface `#FFFFFF` / `#FFFDF6`; border `#F0DFC0` / `#E7D8BD`; heart-empty `#E9DAC4`; text `#5A4327` / `#8A6A45` / `#B08F63`.
- **Test harness:** The repo has no unit-test runner (by design, matching the POC). Per-task verification is `npx tsc --noEmit` + `npm run lint` + the precise manual/boot check stated in the task. Pure logic is isolated into RN-free functions so it *could* be unit-tested later, but adding a runner is out of scope (YAGNI).

### Task 2: Animation primitives

**Files:**
- Create: `src/render/anim.ts`

**Interfaces:**
- Consumes: `react-native-reanimated`.
- Produces:
  - `useSwing(deg: number, periodMs: number): SharedValue<number>` — angle in **radians**, oscillating `-rad..+rad` (ping-pong).
  - `useBob(px: number, periodMs: number): SharedValue<number>` — translateY oscillating `0..-px` (ping-pong).
  - `useSpin(periodMs: number, dir?: 1 | -1): SharedValue<number>` — angle in radians, continuous `0..2π`.

- [ ] **Step 1: Write `src/render/anim.ts`**

```ts
import { useEffect } from 'react';
import {
  useSharedValue, withRepeat, withTiming, Easing, type SharedValue,
} from 'react-native-reanimated';

const DEG = Math.PI / 180;

/** Ping-pong rotation in radians: -deg..+deg over periodMs each half. */
export const useSwing = (deg: number, periodMs: number): SharedValue<number> => {
  const v = useSharedValue(-deg * DEG);
  useEffect(() => {
    v.value = withRepeat(
      withTiming(deg * DEG, { duration: periodMs, easing: Easing.inOut(Easing.ease) }),
      -1, true,
    );
  }, [deg, periodMs, v]);
  return v;
};

/** Ping-pong vertical bob: 0..-px over periodMs each half. */
export const useBob = (px: number, periodMs: number): SharedValue<number> => {
  const v = useSharedValue(0);
  useEffect(() => {
    v.value = withRepeat(
      withTiming(-px, { duration: periodMs, easing: Easing.inOut(Easing.ease) }),
      -1, true,
    );
  }, [px, periodMs, v]);
  return v;
};

/** Continuous rotation 0..2π (dir=-1 reverses). */
export const useSpin = (periodMs: number, dir: 1 | -1 = 1): SharedValue<number> => {
  const v = useSharedValue(0);
  useEffect(() => {
    v.value = withRepeat(
      withTiming(dir * 2 * Math.PI, { duration: periodMs, easing: Easing.linear }),
      -1, false,
    );
  }, [periodMs, dir, v]);
  return v;
};
```

- Consumes: `useSwing`, `useBob` from `src/render/anim.ts`; Skia `Group`, `Circle`, `Path`, `Oval`, `Path.MakeFromSVGString`, `vec`.
- Produces:
  - `useWalkCycle(): { legSwing: SharedValue<number>; armSwing: SharedValue<number>; bobY: SharedValue<number> }`

**Porting recipe (apply to every character; geometry is verbatim from `Foodie.dc.html`):**
- SVG `<circle cx cy r fill>` → Skia `<Circle cx cy r color={fill} />`.
- SVG `<ellipse cx cy rx ry fill [transform=rotate(a x y)]>` → `<Oval rect={rrect? no}>`; use `<Oval x={cx-rx} y={cy-ry} width={2rx} height={2ry} color={fill}/>`, wrapping in `<Group origin={vec(x,y)} transform={[{rotate: a*DEG}]}>` when the SVG had a rotate.
- SVG `<rect x y width height rx fill [transform]>` → `<RoundedRect x y width height r={rx} color={fill}/>` (+ rotate Group if present).
- SVG `<path d=… fill|stroke>` → `const p = Skia.Path.MakeFromSVGString(d)!;` then `<Path path={p} color={fill}/>` (fill) or `<Path path={p} color={stroke} style="stroke" strokeWidth={w} strokeCap="round"/>`.
- Animated groups: wrap the exact `<g style="transform-origin:Xpx Ypx;animation:ff-leg…">` subtrees in `<Group origin={vec(X,Y)} transform={useDerivedValue(() => [{ rotate: legSwing.value }])}>`. The counter-phase leg/arm uses `-value` via a second derived value. Body bob group wraps in `<Group transform={useDerivedValue(() => [{ translateY: bobY.value }])}>`.
- Colors that exist in the theme (cheek pink `#FF9DB0`/`#FFC1CB`, eye `#3B2B20`) may stay as local constants in `palette.ts`; brand-level colors are not used inside characters, so no `useTheme()` needed here — keep the per-character palette in `palette.ts` so a reskin can find them.

import { create } from 'zustand';
import { GameState, GameActions, INITIAL_GAME_STATE, START_LIVES, Who } from '../types/game';

interface GameStore { state: GameState; actions: GameActions; }

export const useGameStore = create<GameStore>((set) => ({
  state: INITIAL_GAME_STATE,
  actions: {
    pick: (who: Who) => set((s) => ({ state: { ...s.state, who } })),
    start: () => set((s) => ({ state: { ...s.state, screen: 'game', lives: START_LIVES, score: 0 } })),
    hitHero: () => set((s) => {
      const lives = Math.max(0, s.state.lives - 1);
      if (lives === 0) return { state: { ...s.state, lives, screen: 'over', best: Math.max(s.state.best, s.state.score) } };
      return { state: { ...s.state, lives } };
    }),
    incrementScore: (points) => set((s) => ({ state: { ...s.state, score: s.state.score + points } })),
    gameOver: () => set((s) => ({ state: { ...s.state, screen: 'over', best: Math.max(s.state.best, s.state.score) } })),
    toSelect: () => set((s) => ({ state: { ...s.state, screen: 'select' } })),
    updateFps: (fps) => set((s) => ({ state: { ...s.state, fps } })),
    reset: () => set((s) => ({ state: { ...INITIAL_GAME_STATE, who: s.state.who, best: s.state.best } })),
  },
}));
